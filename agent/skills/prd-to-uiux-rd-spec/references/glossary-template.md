# 术语表（Glossary）模板

> 用途：冻结“对外名词”与“对内对象/字段”的一致性，降低后续 PRD/页面/组件之间的语义漂移。

日期：YYYY-MM-DD  
状态：Draft/Ready

## 1) 命名规则（建议）

- 对外展示名：中文（面向用户）
- 对内对象名：英文/代码名（面向实现/接口）
- 同义词：必须显式列出并禁止混用（例如：作品/Work、项目/Project）

## 2) 术语表（表格）

| 对外名词（中文） | 对内对象名（英文/代码） | 定义（一句话） | 关键字段/属性（可选） | 备注（边界/非目标） |
|---|---|---|---|---|
| 例：作品 | `Work` | 一个可交付的内容作品的载体 | `work_id`, `title`, `owner_id` | 与 `Project` 的关系 |
| 例：项目 | `Project` | 作品下的一次生产组织单元 | `project_id`, `work_id` | 排期/团队边界 |
| 例：运行/评估记录 | `Run` / `EvalSession` | 一次可回放的 AI 执行 | `status`, `cost`, `elapsed_ms` | 状态机一致性 |

## 3) 关系图（可选）

用 3–6 行文字或简单图示说明：
- `Work` 1..n `Project`
- `Project` 1..n `Run`
- `Run` 产出 `Artifacts`（报告/草稿/导出包）
