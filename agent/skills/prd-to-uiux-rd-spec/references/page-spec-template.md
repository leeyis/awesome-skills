# 页面规格模板（复刻级）

> 复制本模板到 `04_Pages/<PageName>.md`，逐项填写。缺项会导致“看起来像 spec，实际不可落地”。

日期：YYYY-MM-DD  
状态：Draft/Ready

## 1) 页面定位

- 模块：xxx
- 页面：xxx
- 目标用户：xxx（权限/角色）
- 用户目标：xxx

## 2) 布局（Left / Center / Right / Bottom）

- Left（Context）：内容来源、可折叠规则、空态/错误态
- Center（Canvas）：主要操作区、编辑/只读双态、关键主流程
- Right（Command Dock）：指令入口优先、监控项下置
- Bottom（Console，可选）：执行状态/日志，默认折叠策略

## 3) 状态机（必须）

列出：
- 状态集合（例如：就绪/运行/完成/失败/取消）
- 状态转换（触发条件、禁用策略、可恢复策略）
- 失败后用户最小可用路径（例如重试/回退/导出/复制）

## 4) 数据契约（必须）

- 实体与字段：字段名、类型、是否必填、展示规则
- 列表 vs 详情的“信息密度”策略（哪些只在详情展示）
- 排序/过滤：默认与可切换项
- 空态/错误态：文案与 CTA

## 5) 事件与联动（必须）

- hover/click/select → 触发的筛选/高亮/定位
- 定位引用（Location Ref）方案（若需要正文定位）
- 组件间协议（例如 chart click → list filter）

## 6) A11y（必须）

- 键盘路径（tab 顺序）
- ARIA 关键点（role/label/expanded）
- 图表替代视图（若有图表）

## 7) 验收标准（AC）

至少 6 条，覆盖：
- happy path
- 空态/错误态
- 关键联动
- 导出/复制（若涉及）

## 8) 测试计划（离线回归）

写清“怎么验证”：
- 手工回归步骤（无外部依赖）
- 自动化建议（可选）：但本任务不写代码时也必须有手工可复现步骤

## 9) 未尽事宜（显式登记）

- 未来迭代点（P1/P2）
- 未决决策（需要谁拍板）
