# 复刻级就绪检查清单（Replica Readiness）

> 用途：在你“觉得写完了”之前，做一次结构化自查，避免给人类造成不可复刻、不可落地的文档伤害。

## A) 结构自证（打开目录 30 秒内应能判断）

- [ ] 有 `OVERVIEW.md`（总纲 + TODO + 入口）
- [ ] 有 `DOCS_INDEX.md`（每份规格都有入口）
- [ ] 有 `00_SourceInventory/COVERAGE.md`（PRD → 落点映射 + 未覆盖）
- [ ] 有 `01_Foundation/FOUNDATION.md`（跨模块约束冻结）
- [ ] 有 `04_Pages/WorkbenchShell.md`（壳层/插槽契约冻结）
- [ ] 有“术语表/对象模型”冻结结果（至少包含核心对象与关系，可用 `Glossary` 模板产出）

## B) 页面可复刻（抽查任意 2 页）

- [ ] 每页都有状态机（含失败恢复路径）
- [ ] 每页都有数据契约（字段/来源/空态/错误态/排序）
- [ ] 每页都有关键联动（事件 → 结果）说明
- [ ] 每页都有 AC + 测试计划（离线可复现）

## C) 组件可复用

- [ ] “跨模块复用控件”有组件规格（props/event/A11y）
- [ ] “可演进内容”有文档模型/契约（避免写死章节）
- [ ] “图表”明确工作区交互与导出静态化（若涉及）

## D) 交付闭环

- [ ] 仓库 `DOCS_INDEX.md` 已登记
- [ ] `docs/worklog.md` 已记录关键命令/决策/结果
- [ ] （建议）完成一次 `Replica Scorecard` 量化评分，并记录短板项
