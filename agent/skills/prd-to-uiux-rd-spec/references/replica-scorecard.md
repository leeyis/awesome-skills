# 复刻级评分表（Replica Scorecard）

> 用途：轻量、自解释地评估“这套 UI/UX 研发规格是否真的可复刻落地”，避免只看篇幅与模板感。

评分方法：
- 每项 0/1/2 分（0=缺失或不可用；1=有但不完整；2=完整可执行）
- 建议阈值：≥ 16/20 视为“复刻级可落地”；否则必须补齐缺口

| 维度 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 结构自证（目录与入口） | 结构散乱无入口 | 有入口但不成体系 | `OVERVIEW/DOCS_INDEX/COVERAGE/Foundation/Shell` 齐全且可检索 |
| 术语一致性（Glossary） | 无术语/混用严重 | 有术语但未冻结 | 有术语表 + 对外/对内命名一致且被引用 |
| 壳层契约（Shell） | 仅截图/概念 | 有布局但无插槽契约 | Top/Side/Left/Center/Right/Bottom 插槽与规则冻结 |
| 组件契约（Components） | 只在页面里散写 | 有组件说明但无契约 | props/event/A11y/导出策略清晰，可复用 |
| 页面复刻（Pages） | 仅描述体验 | 有页面但缺状态/数据/边界 | 每页有状态机 + 数据契约 + 联动 + 空/错态 |
| 可演进内容（Evolvability） | 写死固定章节 | 允许增减但无机制 | 用 block/model 契约承接，未知降级策略明确 |
| A11y（可访问性） | 未提及 | 仅泛化提及 | 键盘路径/ARIA/替代视图（含图表）可执行 |
| 验收与测试计划 | 无 AC/TestPlan | 有但不可复现 | AC 明确且离线手工回归步骤可复现 |
| 覆盖映射（Coverage） | 无映射 | 映射不完整 | PRD→落点可审计，未覆盖项显式登记 |
| 交付闭环（Index/Worklog） | 无索引/记录 | 有其一 | `DOCS_INDEX` + `worklog` 均更新可追溯 |
